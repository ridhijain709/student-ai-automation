# Folder-by-Folder Restructure Plan

## Goal
Present this repository as a clean client-facing automation portfolio.

## Plan

- `/README.md`  
  Keep business-facing summary and links to demos/docs only.

- `/docs/`  
  Keep planning and client-facing portfolio documentation:
  - `repo-restructure-plan.md`
  - `readme-line-by-line-replacement.md`
  - `github-cleanup-checklist.md`
  - `portfolio-overview.md`
  - `demo-mapping.md`

- `/demos/`  
  Maintain one folder per core automation demo with proof placeholders:
  - `whatsapp-lead-followup/`
  - `content-auto-scheduler/`
  - `faq-auto-reply-bot/`

- `/frontend/src/`  
  Keep portfolio UI focused on the three demos:
  - sidebar navigation points to demo pages
  - dashboard cards highlight business outcomes

- `/backend/`  
  Preserve technical implementation modules; reference only relevant endpoints/services in public docs.
