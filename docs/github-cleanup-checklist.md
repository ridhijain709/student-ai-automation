# GitHub Cleanup Checklist

## Goal
Make the repository look client-ready, business-focused, and easy to understand in under 30 seconds.

---

## Commit Order

### Commit 1: Reposition the repo
- Update README branding
- Remove student/portfolio language
- Make the repo about client-ready AI automations

### Commit 2: Add documentation structure
- Create `docs/`
- Add `portfolio-overview.md`
- Add `demo-mapping.md`
- Add `github-cleanup-checklist.md`

### Commit 3: Add demo structure
- Create `demos/`
- Add one folder per automation demo
- Add placeholder README files for each demo

### Commit 4: Update the UI
- Make the 3 demos the main focus
- Simplify dashboard labels
- Replace generic/internal wording with business outcome wording

### Commit 5: Add proof assets
- Add screenshots
- Add workflow diagrams
- Add Loom demo links

### Commit 6: Final polish
- Verify all links
- Ensure folder names are clean
- Check README formatting
- Remove anything that distracts from the 3 demos

---

## Cleanup Checklist

- [ ] README is client-facing
- [ ] Repo title and descriptions are business-oriented
- [ ] Demo folders exist
- [ ] Demo READMEs explain problem, solution, outcome
- [ ] Screenshots folder exists for each demo
- [ ] Workflow diagrams are included
- [ ] Loom links are included
- [ ] Dashboard reflects the 3 demos
- [ ] Unrelated modules are hidden or archived
- [ ] No student-only language remains
- [ ] Contact or CTA is clear

---

## Final Standard
A client should open the repo and instantly understand:
1. what problem you solve
2. what automations you build
3. what outcome they get
4. how to contact you
