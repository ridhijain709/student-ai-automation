## Folder Structure

Each client folder contains three files:

| File | Contents |
|------|----------|
| `README.md` | Overview: business context, pain points, offer, pricing, outreach messages, next actions |
| `brief.md` | Detailed strategy: pain point analysis, system modules, implementation notes, upsell path |
| `demo-script.md` | Step-by-step demo flow: setup, opening, live demo, close, pricing conversation, objection handling |

---

## How to Use These Materials

### For outreach:
1. Open the client's `README.md`
2. Copy the LinkedIn outreach message (adapt if needed)
3. Send it
4. Use the follow-up message after 3–4 days

### For a demo call:
1. Read the `brief.md` to understand the client's business context
2. Use the `demo-script.md` as your guide
3. Personalise the demo prompts for their specific business before the call

### For a proposal:
1. Use the pain point analysis from `brief.md`
2. Reference the system modules as your deliverables
3. Use the pricing guidance from `README.md`
4. Follow the upsell path in `brief.md` for additional scope

---

## Adding a New Client

Create a new folder in `clients/` using the client's name in kebab-case format (e.g., `new-client-name/`). Add three files following the structure above. Reference an existing client folder as a template.
