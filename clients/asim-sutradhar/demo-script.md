# Asim Sutradhar — Demo Script

## Demo Name: Scalable Business Automation System

## Duration: 20 minutes

## Goal: Demonstrate sophisticated, well-structured automation — not a basic bot, but a system with conditional logic, error handling, and scalability built in.

---

## Note on This Client

Asim understands technology. The demo should be pitched at a peer level — show the architecture and the thinking behind the system, not just the output. Technical credibility matters here.

---

## Setup Before the Demo

- Most sophisticated demo available — ideally showing a multi-step flow
- Make.com scenario with multiple modules and conditional logic visible
- Google Sheet with structured data output
- Optional: show error handling or logging module

---

## Step 1 — Start with Systems Thinking (3 minutes)

Don't start with a basic problem. Start with:

> "Before I show you anything, I want to understand what your current automation setup looks like. Are there tools you're already using — Make.com, Zapier, any API connections? And where are the workflows that are still manual?"

Let them describe their current state. Note:
- What tools are they using?
- Where are the gaps?
- What's most frustrating?

Then say:

> "Most automation setups I see fall into one of two categories — either too basic to be useful at scale, or so complex they break when something small changes. I'll show you what a well-structured approach looks like."

---

## Step 2 — Show the Architecture First (3 minutes)

Before running the demo, walk through the flow diagram or Make.com scenario:

> "Here's the full flow. [Explain each step]. The conditional logic here [point to specific step] handles the cases where X happens. The error branch here ensures nothing gets lost if the API times out. And every action is logged here so there's a full audit trail."

This signals sophistication and builds credibility before the demo even runs.

---

## Step 3 — Run the Demo (7 minutes)

Run the most relevant demo for their business type:

**If product/startup:** Show a lead capture → CRM sync → follow-up flow
**If tech consulting:** Show a client inquiry → classification → routing flow
**If e-commerce:** Show an order inquiry → status check → response flow

Walk through each step:

> "Watch what happens when [trigger]. The system [action 1]. Then [action 2]. Notice that if [condition X] is met, it takes [path A]; otherwise [path B]. The result is [outcome], logged here [show sheet]."

---

## Step 4 — Show the Difference from Basic Automation (3 minutes)

Say:

> "The thing that separates a system that holds up over time from one that breaks is the handling of edge cases. [Show a specific example.] Most basic automation setups ignore this. This one handles it and logs it so you can review and improve."

---

## Step 5 — Close (2 minutes)

> "The way I work with clients like you is: first, we map out which workflows are best suited for automation — based on volume, repetitiveness, and risk. Then we build in phases, starting with the highest-impact item. Each phase is documented and tested before moving to the next."

Ask:

> "What's the one workflow in your business right now that, if it ran reliably without manual effort, would have the highest impact?"

---

## Pricing Conversation

> "Single workflow: ₹8,000–₹15,000 depending on complexity. Multi-workflow systems are ₹20,000–₹35,000. I also offer a monthly retainer for ongoing management and expansion — typically ₹3,000–₹5,000/month."

---

## Objection Handling

**"We have a developer who could build this."**
> "If you have dev capacity, that's great. The question is whether it's the best use of that capacity — and whether the developer's time is better spent on product. Automation consulting is a specialist skill that sits between tech and business process."

**"We've tried automation tools and they're hard to maintain."**
> "That's because most setups aren't built with maintenance in mind. What I deliver includes documentation and is designed to be understood and modified without needing me involved every time something changes."
