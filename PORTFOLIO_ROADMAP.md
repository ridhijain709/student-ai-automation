# Portfolio Delivery Roadmap (Client-Facing)

## Phase 1 — Discovery (Day 0–1)
1. Identify one bottleneck: lead follow-up, content workflow, or response delays.
2. Confirm current process and tools.
3. Select one pilot automation for first implementation.

## Phase 2 — Pilot Build (Day 2–5)
1. Set up data source (sheet/form/inbox feed).
2. Configure automation logic and message templates.
3. Run dry tests with sample records.
4. Share short Loom/video walkthrough for approval.

## Phase 3 — Go Live (Day 6–7)
1. Connect client-owned credentials/accounts.
2. Execute supervised live test.
3. Enable tracking fields for outcomes (sent, replied, pending).
4. Deliver handover SOP (5-minute usage guide).

## Phase 4 — Stabilization (Week 2–4)
1. Monitor edge cases and update templates.
2. Add one optimization (timing rules, better prompts, segmentation).
3. Review outcomes and propose next automation module.

## Suggested Meeting Cadence
- Kickoff: 45 minutes
- Mid-build review: 20 minutes
- Go-live handover: 30 minutes
- Week-2 optimization review: 20 minutes

## Proof Assets for LinkedIn/GitHub
- Before/after workflow screenshot.
- 30–60 second screen demo.
- Short case note: problem → automation → outcome.
